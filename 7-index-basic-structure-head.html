Index

- La primera página siempre tiene que llamarse "index.html".
- El index.html será la página inicial que el servidor buscará, por defecto, al momento de abrir o ejecutar un proyecto.
- Cuando no tenemos una página index.html, nosotros debemos guiar al servidor, manualmente, para indicarle cuál sería nuestra página de inicio. De lo contrario, se visualizará toda la estructura de los directorios de archivos que tenemos en el proyecto.

Si nuestra aplicación web (WebApp) tiene más páginas, como por ejemplo, "contacto.html", "logging.html", etc; estas páginas las vamos a ligar a la única página principal, que vendría a ser "index.html".

  • < !DOCTYPE html >
    Esta etiqueta le indicará al Navegador que la estructura que analizará, a continuación, será de HTML:5, y así es como entenderá todas las etiquetas de HTML:5.
  • < html lang="en">
    "Contenedor principal" / "Contenedor padre", este contenedor llevará todas las etiquetas y todo el contenido que tendrá nuestro proyecto.
    Esta etiqueta llevará un atributo especial que será (lang=" "). El atributo lang es el encargado de decirle al Navegador qué lenguaje tendrá nuestro proyecto, ya sea Inglés, Español, Francés, o cualquiera que sea.
    Un ejemplo del uso del atributo lang, es para que el navegador sepa en qué idioma está la página que visitamos, y si está en un idioma distinto al nuestro, la pueda traducir a nuestro idioma.
    El valor de la etiqueta lang, es determinado por el tipo de idioma que tendrá la página web, por ejemplo, si es una web en español, será "es", lang="es", o si es una página en inglés, será "en", lang="en", o si es una página en francés, será "fr", lang="fr", y así, sucesivamente.
  • < head > < / head >
    La etiqueta "head" es contenedor hermano de la etiqueta "body", porque estas dos etiquetas están en el mismo nivel de la estructura del DOM: Document Object Model.
    Aquí va todo lo que es importante y necesario para el navegador, para que pueda cargar el proyecto tal y como lo diseñamos. Esta sección del código NO ES VISUAL para el usuario final, o sea, todo lo que son dependencias, librerías externas, fuentes, css, etc; todo eso tiene que ir ligado dentro de la etiqueta < head > < / head >. En esta sección mandamos a llamar a fuentes externas.
  • < body > < / body >
    Como ya se mencionó, la etiqueta "body" es contenedor hermano de la etiqueta "head", porque ambos están en el mismo nivel de la estructura del DOM: Document Object Model.
    El usuario final verá todo lo que va dentro de la etiqueta "< body > < / body >", todo lo que es texto, imágenes, videos, animaciones, etc; todo el aspecto interactivo que tendrá nuestro proyecto con el usuario, irá dentro del body.
  • Tabuladores: Son los espacios que debemos dejar al momento de escribir cada etiqueta para generar una simetría cuando esribimos código, para tener una lectura limpia y una comprensión rápida, y el IDE o Editor de Texto nos pueda ayudar a visualizar las estructuras de cada etiqueta, indicándonos su apertura y cierre.
    - Hay etiquetas que tienen apertura y cierre, ejemplo, < body > < / body >, como también hay etiquetas que se autocierran, por ejemplo, < br / >

Una analogía para entender la estructura entre el head y el body, sería como si fuera un cuerpo humano, ya que en head almacenaría todas las conexiones nerviosas que necesita nuestro cuerpo, pero que no son vizuales, y en el body iría todo lo que usamos en el día a día, como nuestras articulaciones, que es claramente visible para el resto de personas.
