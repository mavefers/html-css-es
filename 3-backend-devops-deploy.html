Perfil del Backend

(Servidor)

Backend, en programación, corresponde a "el lado opuesto de un Front-end" en un sitio web o aplicación, ya que el Backend trabaja en el lado del servidor, mientras que el Frontend lo hace del lado del cliente.
- El Backend es el responsable de manejar toda la lógica que hay detrás de una petición dada por el navegador(cliente) hacia el servidor.

Una característica que lo diferencia del Frontend es que no tiene estándares, puesto que tiene varios lenguajes de programación con los que debe trabajar.

Tecnologías

(Lenguajes de Programacion)
  • Python
  • NodeJS (servidor)
  • php
  • Go
  • Rubi
  • Java
  • .NET
Cada uno de estos lenguajes tiene sus propios frameworks como:
  • Django (Python)
  • Laravel (PHP)
  • Rails (Ruby)
  • Express (JavaScript=NodeJS)
  • Spring (Java)
(Recordemos que: Los frameworks nos ayudan a realizar ciertas cosas muy específicas y a construir aplicaciones como más robustas, de manera más rápida)

Un Backend también tiene que ver dónde va a subir su aplicación, o en dónde va a generar el deploy de su aplicación, esto tiene que ver con la Nube.
De aquí sale otro perfil, que se le conoce como DevOps, que son personas que se especializan ya en trabajar con estos servicios de infraestructura en la Nube.
Aún así, el Backend tiene que entender como es que funciona el sistema y cómo puede conectar con la base de datos, para también poder hacer un deploy de su aplicación, conectado con la parte del Frontend.
Infraestructura:

  • Google Cloud
  • aws: Amazon Web Services
  • Digital Ocean
  • HEROKU
  • entre otras...

Con esto podremos subir aplicaciones muy sencillas, como las estáticas, o aplicaciones más complejas, como las dinámicas; y así podremos saber qué tipo de tecnología nos conviene más, dependiendo de nuestro tipo de aplicación y el proyecto que estamos generando.

¿Qué es un deploy?
Deploy es un término famoso entre los desarrolladores web. Puede significar muchas cosas, dependiendo del ambiente y de la tecnología usada.
- Sin embargo, los significados que más se refieren a la práctica y pueden resumir su función, son: implantar, colocar en posición, habilitar para el uso o, simplemente, publicar.

Por último, entramos en bases de datos, que son las encargadas de almacenar toda la información del proyecto.
El Backend también trabaja con base de datos, también tiene que generar esta arquitectura, tiene que poder conectar esta tecnología con el lenguaje o framework que esté utilizando. Para eso existen diferentes bases de datos:
Los principales tipos son:

  • Bases de datos relacionales: MySQL
  • Bases de datos no relacionales: mongoDB
Existen diferentes lenguajes dentro de los grupos de tipos de bases de datos, los cuales se implementan dependiendo del tipo de proyecto y la complejidad que pueda haber.
Se necesita aprender a utilizarlas, para implementarlas en el stack(Backend).
